# Goal
Design dashboards that balance clarity, density, hierarchy, and efficiency for users who need to monitor and act on information.

# Instructions
1. Identify the user's top tasks, decisions, and alerts before arranging panels or charts.
2. Prioritize information hierarchy so the most important metrics, changes, and actions surface first.
3. Use a layout system that supports density without turning the screen into visual noise.
4. Design empty, loading, warning, and error states alongside the ideal data-rich state.
5. Review for scan speed, filter discoverability, table readability, and action clarity.

# Input
A request to design dashboard screens, admin panels, analytics surfaces, or operational product UI.

# Output
Dashboard layouts, components, and UI structure optimized for data visibility and actionability.

# Best Practices
- Use typography, spacing, and color to separate importance levels instead of decorating every card equally.
- Keep filters, bulk actions, and drill-down actions easy to locate.
- Design for real data lengths and edge states, not only polished mock data.
- Preserve accessibility and performance even in dense interfaces.
